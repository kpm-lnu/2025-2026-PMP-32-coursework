# Система ідентифікації відбитків пальців

Біометричний застосунок для розпізнавання відбитків пальців за допомогою екстракції мінуцій та тріангуляції Делоне. Система реєструє користувачів у локальній базі даних SQLite та ідентифікує невідомі відбитки, порівнюючи вектори ознак трикутників, побудованих на скелеті відбитка.

---

## Принцип роботи

1. Зображення відбитка бінаризується та скелетизується.
2. З скелета витягуються мінуційні точки (закінчення гребенів та роздвоєння).
3. На основі мінуцій будується тріангуляція Делоне.
4. Кожен трикутник кодується як вектор ознак (довжини сторін, типи мінуцій, кут орієнтації).
5. При ідентифікації ознаки запиту порівнюються з базою даних з урахуванням геометричних допусків і фільтра домінантного кута.
6. Найкращий збіг повертається, якщо оцінка перевищує заданий поріг.

## Встановлення та запуск

### 1. Склонувати репозиторій

```bash
git clone https://github.com/<ваш-username>/<ваш-репозиторій>.git
```

### 2. Перейти до директорії проєкту

```bash
cd <ваш-репозиторій>
```

### 3. Встановити всі необхідні бібліотеки

Усі залежності можна встановити однією командою:

```bash
pip install opencv-python numpy scikit-image matplotlib scipy
```

> **Порада для VS Code:** відкрийте вбудований термінал через `Terminal → New Terminal` (або `Ctrl + \``) і виконуйте команди там — Python-середовище підхопиться автоматично.

> **Порада для Windows:** якщо команда `pip` не знайдена, спробуйте `python -m pip install ...`

### 4. Підготувати зображення відбитків

Розмістіть файли відбитків (формат `.tif`) у **тій самій директорії**, що й `main.py`. За замовчуванням скрипт використовує конвенцію іменування баз даних [FVC](http://bias.csr.unibo.it/fvc2002/):

```
101_1.tif   ← зображення для реєстрації суб'єкта 101
101_2.tif   ← зображення для ідентифікації суб'єкта 101
102_1.tif
102_2.tif
...
```

> Суфікс `_1` використовується для реєстрації, `_2` — для тестування ідентифікації. Можна використовувати будь-які напівтонові `.tif`-зображення — просто оновіть імена файлів у скрипті.

### 5. Запустити програму

```bash
python fingerprint.py
```

## База даних

Застосунок використовує локальну базу даних **SQLite** (`biometrics.db`), яка створюється автоматично при першому запуску. Зберігається наступне:

| Колонка         | Опис                                                  |
|-----------------|-------------------------------------------------------|
| `id`            | Автоінкрементний первинний ключ                       |
| `name`          | Унікальне ім'я користувача (напр. `"Суб'єкт 101"`)   |
| `features_json` | JSON-серіалізовані ознаки трикутників Делоне          |
| `image_path`    | Шлях до оригінального зображення реєстрації          |

> **Увага:** База даних **перестворюється при кожному запуску** (через виклик `init_db()`). Щоб зберегти зареєстрованих користувачів між запусками — закоментуйте виклик `init_db()` після першого запуску.

## Налаштування

У блоці `__main__` можна змінити:

```python
THRESHOLD = 19.0  # Мінімальна оцінка (%) для підтвердження збігу
```

Менше значення → м'якше порівняння (більше хибних спрацювань).  
Більше значення → суворіше порівняння (більше хибних відмов).

---

## Що відбувається при кожному запуску

```
1. Ініціалізація (перестворення) бази даних SQLite
2. Реєстрація 4 суб'єктів за їхніми зображеннями _1.tif
3. Тест 1 — той самий палець, інший скан: запит 104_2.tif → очікується збіг з "Суб'єкт 104"
4. Тест 2 — інша людина: запит 102_2.tif → очікується відмова або низька оцінка
5. Відкривається вікно matplotlib із скелетом, мінуціями, тріангуляцією та збіжними трикутниками
```

Для кожного тесту відкривається вікно порівняння:
- **Верхній рядок:** мінуційні точки, тріангуляція Делоне, центроїди збіжних трикутників (позначено `+`)
- **Нижній рядок:** бінарні скелети обох відбитків
